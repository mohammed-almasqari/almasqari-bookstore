# 📚 متجر الكتب الرقمية — محمد المسقري

متجر إلكتروني احترافي لبيع وتوزيع الكتب الرقمية، مع لوحة تحكم عربية كاملة للمؤلف،
دفع آمن عبر **PayPal**، تسليم تلقائي بالبريد عبر **Resend**، وروابط تحميل مشفّرة ومؤقتة.

مبني بتقنية **Next.js** و **PostgreSQL** و **Prisma**، وجاهز للنشر عبر **Docker / Coolify**.

---

## ✨ المزايا

- **واجهة متجر عصرية** بالعربية (RTL) متجاوبة بالكامل مع الجوال والحاسوب، ببطاقات كتب أنيقة.
- **صفحة كتب مجانية** بآلية تأكيد بريد مزدوجة (Double Opt-in): يسجّل الزائر اسمه وبريده، يؤكّد بريده بضغطة، فيصله الكتاب فورًا.
- **شراء عبر PayPal**: دفع آمن، إيصال شراء بالبريد، ورابط تحميل فوري.
- **روابط تحميل آمنة**: رموز عشوائية، صلاحية زمنية، وحد أقصى لعدد مرّات التحميل.
- **بريد احترافي عبر Resend** بثلاثة قوالب مصمّمة: تأكيد التسجيل، تسليم الكتاب، وإيصال الشراء.
- **لوحة تحكم للمؤلف**: إحصاءات، إدارة الكتب (إضافة/تعديل/حذف ورفع الملفات والأغلفة)، عرض الطلبات والمشتركين.
- **قاعدة بيانات PostgreSQL داخل Docker** عبر `docker-compose`.

## 🚀 التشغيل المحلي (للتطوير)

> المتطلبات: Node.js 20+ و PostgreSQL (أو Docker).

```bash
# 1) ثبّت الاعتماديات
npm install

# 2) جهّز ملف البيئة
cp .env.example .env
#   ثم عدّل القيم (DATABASE_URL على الأقل)

# 3) أنشئ الجداول وابذر البيانات (حساب المدير + كتاب مجاني)
npm run db:push
npm run db:seed

# 4) شغّل الخادم
npm run dev
```

- المتجر: <http://localhost:3000>
- لوحة التحكم: <http://localhost:3000/admin> (بيانات الدخول من `ADMIN_EMAIL` و `ADMIN_PASSWORD`)

### التشغيل عبر Docker محليًا

```bash
cp .env.example .env   # عدّل القيم
docker compose up --build
```

يشغّل هذا قاعدة البيانات والتطبيق معًا. عند أول إقلاع تُنشأ الجداول وتُبذر البيانات تلقائيًا.

## 🔑 المتغيّرات السرّية (مهم)

كل الأسرار في ملف `.env` (انظر `.env.example`). **لا تُرفع `.env` إلى GitHub أبدًا** — وهو مستثنى في `.gitignore`.

### 1) PayPal
1. ادخل <https://developer.paypal.com/dashboard/applications>.
2. أنشئ تطبيقًا (Sandbox للتجارب، Live للإنتاج).
3. انسخ **Client ID** و **Secret** إلى:
   - `NEXT_PUBLIC_PAYPAL_CLIENT_ID`
   - `PAYPAL_CLIENT_SECRET`
   - `PAYPAL_ENV=sandbox` أو `live`

### 2) Resend (البريد)
1. أنشئ حسابًا في <https://resend.com> وأضف **مفتاح API** في `RESEND_API_KEY`.
2. **وثّق نطاقك** (Domains → Add Domain) بإضافة سجلات DNS التي يعطيك إياها Resend في هوستنجر.
3. اضبط `EMAIL_FROM` ببريد على نطاقك الموثّق، مثل: `مكتبة محمد المسقري <no-reply@yourdomain.com>`.

### 3) مفتاح الجلسات
ولّد قيمة عشوائية طويلة لـ `AUTH_SECRET`:
```bash
openssl rand -base64 48
```

## 🌊 النشر على Coolify

> Coolify منصّة استضافة ذاتية تدير حاويات Docker. الخطوات:

1. **ارفع المشروع إلى GitHub** (انظر القسم التالي).
2. في Coolify: **New Resource → Public/Private Repository**، واختر مستودعك.
3. **Build Pack**: اختر **Docker Compose** (المشروع يحوي `docker-compose.yml` جاهزًا).
   - أو اختر **Dockerfile** وأضف قاعدة بيانات PostgreSQL كمورد منفصل.
4. **متغيرات البيئة**: أضف كل متغيّرات `.env` في تبويب *Environment Variables* داخل Coolify
   (ضع الأسرار هناك — لا داخل المستودع).
5. **التخزين الدائم (Persistent Storage)**: تأكّد من بقاء حجمي `db_data` و `uploads_data`
   حتى لا تُفقد قاعدة البيانات ولا الكتب المرفوعة عند إعادة النشر.
6. **المنفذ**: التطبيق يستمع على `3000`.
7. **فحص الصحة**: استخدم المسار `/api/health`.
8. اضغط **Deploy**. عند أول إقلاع تُنشأ الجداول ويُبذر حساب المدير تلقائيًا.

> رفع ملفات كبيرة: إن استخدمت بروكسي أمامي، ارفع حد حجم الطلب (مثلاً `client_max_body_size 60M`).

## 🌐 ربط دومين هوستنجر الفرعي

لربط نطاق فرعي مثل `books.yourdomain.com`:

1. في **Coolify**: افتح خدمة التطبيق → تبويب **Domains** → أضف `https://books.yourdomain.com`.
   (سيتولّى Coolify إصدار شهادة SSL تلقائيًا عبر Let's Encrypt.)
2. في **هوستنجر** (hPanel → Domains → DNS / Nameservers → DNS Records):
   - أضف سجلًا من نوع **A**:
     - **Name/Host**: `books`
     - **Points to**: عنوان IP الخادم الذي يعمل عليه Coolify
     - **TTL**: الافتراضي
   - (إن كان خادمك يستخدم IPv6 أضف سجل **AAAA** كذلك.)
3. انتظر انتشار DNS (دقائق إلى ساعات)، ثم تأكّد أن `NEXT_PUBLIC_SITE_URL=https://books.yourdomain.com`
   في متغيرات البيئة، وأعد النشر.

## 🧑‍💼 أول تشغيل وإدارة الكتب

1. ادخل لوحة التحكم: `https://your-domain/admin` ببيانات `ADMIN_EMAIL` و `ADMIN_PASSWORD`.
2. **غيّر كلمة المرور**: عدّل `ADMIN_PASSWORD` في البيئة ثم أعد التشغيل (يُحدَّث الحساب تلقائيًا).
3. من **الكتب → إضافة كتاب**: ارفع ملف PDF والغلاف، وحدّد السعر أو اجعله مجانيًا.
4. كتاب «لن يخدعوك بعد اليوم» مُهيّأ مسبقًا ككتاب مجاني — ما عليك إلا رفع ملف الـ PDF والغلاف له.

## 📁 بنية المشروع

```
store/
├─ app/
│  ├─ (site)/            صفحات المتجر (الرئيسية، المكتبة، الكتاب، المجاني، التأكيد)
│  ├─ admin/             لوحة التحكم (دخول + لوحة + كتب + طلبات + مشتركون)
│  └─ api/               مسارات الخادم (free, checkout, download, cover, admin, health)
├─ components/           مكونات الواجهة (بطاقات، نماذج، أيقونات، لوحة التحكم)
├─ lib/                  المنطق (db, auth, paypal, email, storage, tokens, ...)
├─ prisma/               مخطط قاعدة البيانات + سكربت البذور
├─ Dockerfile           صورة الإنتاج
├─ docker-compose.yml    التطبيق + قاعدة البيانات
└─ .env.example          نموذج المتغيّرات السرّية
```
